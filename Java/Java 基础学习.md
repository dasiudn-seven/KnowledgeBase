# Java 基础学习

## Java基础语法回顾

### 数据类型（八大基本类型 vs 引用类型）

#### 一、8 大基本数据类型（Primitive Types）

Java 内置的、最基础的类型，**不是对象**，存的是**真实值**。

##### **分类 + 取值范围 + 占用空间**

| 类型分类   | 数据类型   | 占用字节 | 取值范围             | 默认值   |
| ---------- | ---------- | -------- | -------------------- | -------- |
| **整数型** | byte       | 1        | -128 ~ 127           | 0        |
|            | short      | 2        | -32768 ~ 32767       | 0        |
|            | **int**    | 4        | -2¹⁴ ~ 2³¹-1         | 0        |
|            | long       | 8        | 超大范围             | 0L       |
| **浮点型** | float      | 4        | 单精度小数           | 0.0f     |
|            | **double** | 8        | 双精度小数           | 0.0d     |
| **字符型** | char       | 2        | 0 ~ 65535（Unicode） | '\u0000' |
| **布尔型** | boolean    | 1 位     | true / false         | false    |

##### **核心特点**

1. **存的是值本身**，直接放在栈内存
2. **没有属性和方法**，不能调用 `.`
3. **效率极高**，是 Java 性能基础
4. 有**包装类**：`Integer、Long、Boolean` 等

#### 二、引用数据类型（Reference Types）

除了 8 大基本类型，**剩下的全是引用类型**。

##### 常见类型

- 类（`String、自定义类、Object`）
- 接口
- 数组（`int[]、String[]`）
- 枚举

##### 核心特点

1. **存的是地址**（指向堆内存里的对象）
2. **默认值是 null**
3. 可以调用**方法、访问属性**
4. 由 `new` 关键字创建对象

#### 三、最核心区别

| 对比项     | 基本类型              | 引用类型                           |
| ---------- | --------------------- | ---------------------------------- |
| 存储内容   | **真实值**            | **内存地址**                       |
| 内存位置   | 栈内存                | 地址在栈，对象在堆                 |
| 默认值     | 数字 0、false、空字符 | **null**                           |
| 是否为对象 | 不是                  | 是                                 |
| 占用空间   | 固定小字节            | 不固定                             |
| 赋值方式   | 直接赋值              | 多为 new 创建                      |
| 比较方式   | `==` 比较**值**       | `==` 比较**地址**，equals 比较内容 |

#### 四、代码示例一看就懂

##### 1、基本类型

```java
int a = 10;  // 栈里直接存 10
int b = a;   // 把 10 复制一份给 b
a = 20;
System.out.println(b);  // 输出 10（互不影响）
```

##### 2、引用类型

```java
String s1 = new String("hello");  // 栈存地址，堆存对象
String s2 = s1;                   // s2 复制了地址，指向同一个对象
System.out.println(s1 == s2);     // true（地址相同）
```

#### 五、注意点

##### 1、**int 和 Integer 区别？**

int 是基本类型，存值；Integer 是引用类型，存地址，允许 null。

#####  2、**== 和 equals 区别？**

基本类型用 `==` 比**值**；

引用类型用 `==` 比**地址**，用 `equals()` 比**内容**。

##### 3、**基本类型一定比引用类型快吗？**

绝大多数场景：是。基本类型无对象开销，速度更快。

#### 总结

* **8 大基本类型**：`byte、short、int、long、float、double、char、boolean`，存**值**，高效。

* **引用类型**：类、接口、数组等，存**地址**，是对象。

* 核心区别：**存值 vs 存地址**。



### Java 变量作用域（Scope）

变量作用域就是：**变量能在代码的哪个范围内被使用**，出了这个范围就报错，Java 里一共就 **3 种作用域**，非常好记。

---

#### 一、3 种变量作用域

##### 1. 局部变量（方法内）

- **定义**：写在**方法、构造方法、代码块**内部的变量
- **作用域**：从定义开始，到当前大括号 `{}` 结束
- **特点**：
  - 必须**手动赋值**才能用（没有默认值）
  - 只能在当前方法 / 代码块里用
  - 方法执行完就销毁

```java
public void test() {
    int num = 10; // 局部变量
    System.out.println(num); // 可以用
}
// 出了大括号就不能用了
```

##### 2. 成员变量（类内、方法外）

- **定义**：写在**类里面、方法外面**的变量
- **作用域**：整个类里都能用（所有方法都能访问）
- **特点**：
  - 有**默认值**（int=0，boolean=false，引用类型 = null）
  - 随着对象创建而存在，对象销毁才消失

```java
public class Person {
    String name; // 成员变量（有默认值 null）
    int age;     // 成员变量（有默认值 0）

    public void show() {
        System.out.println(name); // 能用
    }
}
```

##### 3. 类变量（static 静态变量）

- **定义**：带 `static` 关键字的成员变量
- **作用域**：**整个类**都能用，甚至不用创建对象
- **特点**：
  - 全类**共享一份**
  - 类加载就存在，程序结束才销毁

```java
public class Student {
    static String school = "第一中学"; // 类变量
}
// 使用：Student.school 直接访问
```

### Java 流程控制

流程控制就是**控制代码执行顺序**，Java 只有 **3 大类**：顺序、分支、循环，所有逻辑都靠它们实现。

---

#### 一、三大流程结构

1. **顺序结构**：从上到下一行行执行（默认）
2. **分支结构**：满足条件才执行（if、switch）
3. **循环结构**：重复执行某段代码（for、while、do-while）

---

#### 二、分支结构（条件判断）

##### 1、 if 系列（最常用）

支持**任意条件**（>、<、==、!=、&&、||）

```java
// 单分支
if(条件) { 执行代码 }

// 双分支
if(条件) { ... } else { ... }

// 多分支
if(条件1) {
} else if(条件2) {
} else {
}
```

##### 2、switch 语句

只能匹配：**byte、short、int、char、String、枚举**

**不支持 > < 这种范围判断**

```java
switch(变量) {
    case 值1: 代码; break;
    case 值2: 代码; break;
    default: 都不匹配时执行;
}
```

⚠️ 注意：**必须加 break**，否则会穿透执行下面代码

#### 三、循环结构（重复执行）

##### **1、for 循环（固定次数，最常用）**

```java
// 打印 1~10
for(int i=1; i<=10; i++){
    System.out.println(i);
}
```

##### 2、while 循环（先判断，再执行）

```java
// 打印 1~10
int i =1;
while(i <= 10) {
		System.out.println(i);
}
```

##### 3、 do-while 循环（先执行一次，再判断）

```java
// 打印 1~10
int i =1;
do {
    System.out.println(i);
} while(i > 10);
```

特点：**至少执行一次**

#### 四、循环控制关键字

1. **break**：跳出**整个循环 /switch**
2. **continue**：跳过**本次循环**，直接进入下一次
3. **return**：直接结束**整个方法**

#### 五、特殊点

1. **if 和 switch 怎么选？**

   - 条件是**范围、复杂判断** → 用 if

   - 条件是**固定值匹配** → 用 switch

2. **for、while、do-while 区别？**

   - for：知道循环次数

   - while：不知道次数，先判断

   - do-while：**至少执行一次**

3. **break 和 continue 区别？**

   - break：直接结束循环

   - continue：跳过这一圈，继续下一圈

#### 五、总结

- **分支**：if（万能）、switch（固定值）
- **循环**：for（次数）、while（条件）、do-while（至少一次）
- **控制**：break（跳出）、continue（跳过）

### 数组声明与初始化

数组就是**固定长度、相同类型**的数据容器，**长度一旦定义不可改变**。我给你整理最常用、最清晰的版本。

---

#### 一、数组声明（2 种写法）

**先声明，才能用**

##### 1. 推荐写法（常用）

```java
// 整型数组
int[] arr;

// 字符串数组
String[] names;

// 浮点型数组
double[] nums;
```

##### 2. 不推荐写法（C 语言风格）

```java
int arr[]; // 不推荐，可读性差
```

#### 二、数组初始化

##### 1、动态初始化（指定长度，默认值）

先指定**长度**，系统自动赋默认值：

- 整数：`0`
- 浮点：`0.0`
- 布尔：`false`
- 引用类型：`null`

```java
// 格式：数据类型[] 数组名 = new 数据类型[长度];
int[] arr = new int[5]; // 长度为5，默认值都是0
```

##### 2、静态初始化（指定内容，自动算长度）

**直接赋值内容**，不用写长度

```java
// 标准格式
int[] arr = new int[]{10, 20, 30, 40};

// 简化格式（最常用！）
int[] arr = {10, 20, 30, 40};
```

##### 3、先声明，再初始化

```java
int[] arr;
arr = new int[]{1,2,3}; // 注意：简化版{}不能单独用
```

#### 三、数组使用（取值、赋值、长度）

```java
// 1. 创建数组
int[] arr = {10, 20, 30};

// 2. 获取元素（下标从 0 开始）
System.out.println(arr[0]); // 10

// 3. 修改元素
arr[1] = 99;

// 4. 获取数组长度
System.out.println(arr.length); // 3
```

#### 四、高频注意事项（坑点）

1. **下标越界**：访问 `arr[100]` 会报错 `ArrayIndexOutOfBoundsException`
2. **空指针**：数组为 `null` 时访问元素会报错
3. **长度固定**：数组一旦创建，**不能改长度**
4. **{} 只能在声明时用**，单独赋值必须用 `new int[]{}`

#### 五、一句话总结

- **动态初始化**：`new int[5]` → 定长度，默认值
- **静态初始化**：`{1,2,3}` → 定内容，自动长度
- **下标从 0 开始**，**长度不可变**

#### 总结

- 声明：`int[] arr;`
- 初始化 3 种：
  1. `new int[5]`
  2. `new int[]{1,2,3}`
  3. `{1,2,3}`（最常用）
- 使用：`arr[0]`、`arr.length`

## 面向对象

### 类和对象的概念

#### 一、一句话记住

- **类（Class）**：是**设计图、模板**
- **对象（Object）**：是根据设计图**造出来的实例**

> 例子：
>
> **手机设计图 = 类**
>
> **你手里的真机 = 对象**

#### 二、类（Class）是什么？

类是**具有相同特征和行为的事物的抽象**。

一个类里包含两样东西：

1. **属性（特征）**：名字、年龄、颜色…
2. **方法（行为）**：吃饭、跑步、说话…

##### 类的写法

```java
// 类：模板
public class Person {
    // 1. 属性（成员变量）
    String name;
    int age;

    // 2. 方法（行为）
    public void eat() {
        System.out.println("吃饭");
    }
}
```

#### 三、对象（Object）是什么？

对象是**类的具体实例**。

- 类是抽象的
- 对象是真实的、可使用的

##### 创建对象 + 使用

```java
public class Test {
    public static void main(String[] args) {
        // 1. 创建对象（根据类造实例）
        Person p = new Person();

        // 2. 使用属性
        p.name = "张三";
        p.age = 20;

        // 3. 调用方法
        p.eat();
    }
}
```

#### 四、类 vs 对象 核心区别

| 类（Class）    | 对象（Object）         |
| -------------- | ---------------------- |
| 抽象的模板     | 具体的实例             |
| 不存在于内存   | 占用内存               |
| 定义属性和方法 | 拥有属性值，能执行方法 |
| 只有一个       | 可以创建无数个         |

#### 五、超通俗比喻

- **类 = 汽车图纸**
- **对象 = 宝马、奔驰、特斯拉**

图纸不能开，**造出来的车才能开**。

同理：

类不能直接用，**必须创建对象才能用**。

#### 六、面试必背

1. **什么是类？**

   类是对象的模板，描述一类事物的属性和行为。

2. **什么是对象？**

   对象是类的实例，是具体存在的实体。

3. **关系？**

   **类是对象的抽象，对象是类的实例。**

#### 总结

- **类**：模板，抽象，定义属性 + 方法
- **对象**：实例，具体，`new` 出来，能使用
- **核心口诀**：**类是模板，对象是实例**

### 构造方法 vs 构造代码块 vs 静态代码块执行顺序

#### 一、先搞清楚三个东西是什么

1. **静态代码块**：`static{}`，加载类时执行
2. **构造代码块**：`{}`，创建对象时执行
3. **构造方法**：`类名(){}`，创建对象时执行

#### 二、最终执行顺序

**静态代码块 → 构造代码块 → 构造方法**

##### 重点规律

1. **静态代码块**：**只执行 1 次**（类第一次加载时）
2. **构造代码块**：**每创建一个对象就执行 1 次**
3. **构造方法**：**每创建一个对象就执行 1 次**

#### 三、代码一看就懂

```java
public class Person {
    // 1. 静态代码块
    static {
        System.out.println("静态代码块执行");
    }

    // 2. 构造代码块
    {
        System.out.println("构造代码块执行");
    }

    // 3. 构造方法
    public Person() {
        System.out.println("构造方法执行");
    }

    public static void main(String[] args) {
        new Person(); // 创建第一个对象
        System.out.println("--------");
        new Person(); // 创建第二个对象
    }
}
```

**输出结果**

```txt
静态代码块执行  ← 只执行1次
构造代码块执行
构造方法执行
--------
构造代码块执行  ← 静态不再执行
构造方法执行
```

#### 四、带继承的执行顺序

如果有**父类 + 子类**，顺序是：

##### **父类静态 → 子类静态 → 父类构造代码块 → 父类构造方法 → 子类构造代码块 → 子类构造方法**

#### 五、一句话终极口诀

**静先动后，先父后子，静态只一次，构造次次走。**

#### 总结

1. 执行顺序：**静态代码块 > 构造代码块 > 构造方法**
2. 静态：**类加载执行，只一次**
3. 构造代码块 & 构造方法：**每 new 一次执行一次**
4. 继承：**先父后子**

### 方法重载 vs 方法重写

#### 一、一句话区分

- **重载（Overload）**：同一个类里，**方法名相同，参数不同**
- **重写（Override）**：子类里，**重写父类的方法**（方法签名完全一样）

#### 二、核心区别表

| 对比项       | 重载（Overload）                 | 重写（Override）       |
| ------------ | -------------------------------- | ---------------------- |
| **位置**     | **同一个类**中                   | **子类 + 父类**之间    |
| **方法名**   | **必须相同**                     | **必须相同**           |
| **参数列表** | **必须不同**（个数、类型、顺序） | **必须完全相同**       |
| **返回值**   | 可以不同                         | 必须相同（或子类类型） |
| **修饰符**   | 随便改                           | 子类权限**不能更严格** |
| **异常**     | 随便改                           | 不能抛出更大异常       |
| **关键字**   | 无                               | `@Override`（推荐）    |

#### 三、代码示例

##### 1、方法重载（同一个类）

```java
public class Test {
    // 方法1
    public void add(int a) {}

    // 重载：参数个数不同
    public void add(int a, int b) {}

    // 重载：参数类型不同
    public void add(double a) {}
}
```

**口诀：同名不同参，我就是重载。**

##### 2、方法重写（子类重写父类）

```java
// 父类
class Father {
    public void say() {
        System.out.println("父类说话");
    }
}

// 子类
class Son extends Father {
    // 重写：方法签名完全一样
    @Override
    public void say() {
        System.out.println("子类说话");
    }
}
```

**口诀：子类改父法，同名又同参。**

#### 四、最关键的 3 条规则

1. **重载**：看**参数**，参数不同就是重载
2. **重写**：看**继承**，子类完全复制父类方法并改实现
3. **重写权限**：子类方法不能比父类**更私有**
   - 父类 public → 子类只能 public
   - 父类 protected → 子类可以 protected /public

#### 五、终极记忆口诀

- **重载：同类同名不同参，编译时决定**
- **重写：子类继承改方法，运行时多态**

#### 总结

- **重载**：同一个类里，方法名相同，**参数不同**
- **重写**：子类里，**完全复制父类方法**并修改实现
- 核心：**重载看参数，重写看继承**

### this /super 关键字用法

这两个是**指代对象**的关键字，专门用来解决**同名变量、方法、构造方法**的调用问题，用法非常固定，背会就不会错！

---

#### 一、核心口诀

* **this**：指向**当前对象自己**

* **super**：指向**父类对象**

#### 二、this 关键字 3 大用法

作用：**访问本类的东西**

##### 1、 区分局部变量和成员变量（最常用）

当**局部变量和成员变量同名**时，用 `this.` 代表**成员变量**。

```java
public class Person {
    String name; // 成员变量

    public void setName(String name) { // 局部变量（同名）
        // 等号左边：成员变量
        // 等号右边：局部变量
        this.name = name; 
    }
}
```

##### 2、调用本类的其他方法

```java
public void eat() {}
public void run() {
    this.eat(); // 调用本类的eat()
}
```

##### 3、调用本类的其他构造方法

必须放在**第一行**！

```java
public Person() {
    // 无参构造
}
public Person(String name) {
    this(); // 调用本类无参构造
    this.name = name;
}
```

#### 三、super 关键字 3 大用法

作用：**访问父类的东西**（解决继承冲突）

##### 1、调用父类的成员变量

```java
class Father {
    int age = 50;
}
class Son extends Father {
    int age = 20;
    
    public void show() {
        System.out.println(this.age); // 20（自己的）
        System.out.println(super.age); // 50（父类的）
    }
}
```

##### 2、调用父类的成员方法

```java
@Override
public void eat() {
    super.eat(); // 先执行父类的eat()
    System.out.println("子类吃饭");
}
```

##### 3、调用父类的构造方法

必须放在**第一行**！

```java
public Son() {
    super(); // 调用父类无参构造（默认隐藏，不写也会自动执行）
}
```

#### 四、最核心区别

| 关键字    | 代表对象   | 用法                     |
| --------- | ---------- | ------------------------ |
| **this**  | 当前类自己 | 访问本类变量、方法、构造 |
| **super** | 父类       | 访问父类变量、方法、构造 |

#### 五、2 个必须记住的规则

1. **this (...) 和 super (...) 都必须在构造方法第一行**

   所以：**一个构造方法里，this 和 super 不能同时写！**

2. **子类构造方法默认第一行有 super ()**

   如果你不写，系统会自动送你一行 `super();` 调用父类无参构造。

#### 六、一句话总结

- **this.**：找**自己**的成员
- **super.**：找**爸爸**的成员
- **this()**：调用自己构造
- **super()**：调用爸爸构造

#### 总结

- **this**：访问**本类** → 变量、方法、构造
- **super**：访问**父类** → 变量、方法、构造
- 核心：**this 是我，super 是爹**

### 四大访问权限修饰符范围（public/protected/default/private）

核心就是：**控制一个类 / 变量 / 方法能被谁访问、不能被谁访问**。

---

#### 一、四大权限（从小到大）

1. **private**  私有
2. **default**  默认（包权限，**什么都不写**）
3. **protected** 受保护
4. **public**  公共

#### 二、访问范围终极表

| 修饰符        | 本类 | 同包 | 子类（不同包） | 任意位置 |
| ------------- | ---- | ---- | -------------- | -------- |
| **private**   | ✅    | ❌    | ❌              | ❌        |
| **default**   | ✅    | ✅    | ❌              | ❌        |
| **protected** | ✅    | ✅    | ✅              | ❌        |
| **public**    | ✅    | ✅    | ✅              | ✅        |

#### 三、一句话通俗解释

1. **private（私）**：只有**自己能用**，亲儿子都不行
2. **default（包）**：**同包一家人**能用，外人不行
3. **protected（护）**：同包 + ** 亲儿子（子类）** 能用
4. **public（公）**：**全天下都能用**

#### 四、权限范围大小（必背）

**private < default < protected < public**

#### 五、高频坑点

1. **default 不是关键字**，就是**不写修饰符**
2. **protected 在不同包的非子类中不能访问**
3. 类（外部类）只能用：**public、default**

#### 六、记忆口诀

**私为本，包为同，护为子，公为全。**

- 私：只有本类
- 包：同包可用
- 护：子类可用
- 公：全局可用

#### 总结

- 范围从小到大：**private → default → protected → public**
- 核心：**public 最大，private 最小**

## 面向对象核心

### 继承：Java是单继承，理解为什么

#### 一、先记住结论

**Java 只支持：单继承 + 多层继承，不支持多继承。**

- 单继承：一个子类 **只能有 1 个父类**
- 多继承：一个子类 **想同时有 2 个父类**（Java 禁止）

```java
// Java 允许（单继承）
class Son extends Father { }

// Java 报错！不允许多继承
class Son extends Father, Mother { }
```

#### 二、为什么不能多继承？—— 因为会产生**二义性（致命问题）**

核心原因只有一个：

**如果两个父类有同名方法，子类不知道该用谁的！**

##### 举个经典例子：菱形问题 / 死亡钻石问题

```java
class Father {
    public void say() { System.out.println("我是爸爸"); }
}
class Mother {
    public void say() { System.out.println("我是妈妈"); }
}

// 如果允许多继承
class Son extends Father, Mother {
    // 问：调用 son.say() 时，执行谁的？
    // 编译器彻底懵了！
}
```

👉 **这就叫二义性，Java 为了安全、简单，直接禁止多继承！**

#### 三、Java 设计者的思路

1. **多继承太复杂，容易出 bug**

   方法冲突、变量冲突、构造方法冲突，问题极多。

2. **Java 追求：简单、安全、高效**

   宁愿少一个功能，也不让程序出问题。

3. **用接口（interface）弥补功能**

   Java 虽然不能**多继承类**，

   但可以**多实现接口**，完全能替代多继承的功能，还不会冲突。

```java
// Java 允许：一个类实现多个接口（完美替代多继承）
class Son implements Father, Mother { }
```

#### 四、一句话总结

**Java 是单继承，为了避免多个父类出现同名方法或变量时，产生二义性，让程序更安全、更简单。**

#### 五、补充：Java 继承的正确姿势（多层继承）

虽然不能同时继承 2 个父类，但可以**多层继承**：

```java
GrandFather
    ↑
Father
    ↑
Son
```

这是完全允许的，也不会有冲突。

#### 总结

1. **Java 单继承**：一个类只能有一个直接父类
2. **原因**：防止**多继承带来的二义性**
3. **替代方案**：**多实现接口**（既能实现多继承功能，又安全）

### 多态：编译看左边，运行看右边

#### 一、先搞懂：什么是多态？

**父类引用指向子类对象**

```java
// 格式：父类 引用名 = new 子类();
Animal a = new Cat(); 
```

* 左边 `Animal a`：编译类型（父类）
* 右边 `new Cat()`：运行类型（子类）

#### 二、终极口诀：**编译看左边，运行看右边**

##### 1、编译看左边（能不能调用？）

- **看父类有没有这个方法**
- 父类有 → 能调用
- 父类没有 → **编译报错**

##### 2、 运行看右边（执行谁的？）

- **真正运行执行子类重写的方法**
- 子类重写了 → 执行子类
- 子类没重写 → 执行父类

#### 三、代码一看就懂

```java
// 父类
class Animal {
    public void eat() {
        System.out.println("动物吃饭");
    }
}

// 子类
class Cat extends Animal {
    // 重写
    @Override
    public void eat() {
        System.out.println("猫吃鱼");
    }
}

public class Test {
    public static void main(String[] args) {
        // 多态：父类引用 指向 子类对象
        Animal a = new Cat();
        
        // 编译：看左边 Animal 有没有 eat() → 有，编译通过
        // 运行：看右边 Cat 重写了 eat() → 执行猫的
        a.eat(); // 输出：猫吃鱼
    }
}
```

#### 四、变量怎么看？（重要坑点）

**变量不参与重写！变量永远看左边！**

```java
class Animal {
    String name = "动物";
}
class Cat extends Animal {
    String name = "小猫";
}

public class Test {
    public static void main(String[] args) {
        Animal a = new Cat();
        System.out.println(a.name); // 输出：动物（永远看左边）
    }
}
```

#### 五、一句话总结（背会就满分）

1. **方法**：编译看左边，运行看右边（多态）
2. **变量**：永远看左边（不参与多态）

#### 六、注意事项

**多态的前提：**

1. 必须有**继承 / 实现**
2. 必须有**方法重写**
3. 必须是**父类引用指向子类对象**

#### 总结

- **编译看左边**：决定能不能调用（父类）
- **运行看右边**：决定执行谁的（子类）
- **方法多态，变量看左**

### 多态向上转型 & 向下转型

#### 一、先记住核心比喻

父类 = **人类**

子类 = **程序员**

- **向上转型**：程序员 → 人类（自动转，安全）
- **向下转型**：人类 → 程序员（强制转，有风险）

#### 二、向上转型（自动转，安全）

##### 1. 定义

**子类对象 赋值给 父类引用**

（小范围转大范围，自动完成）

##### 2. 格式

```java
父类 引用 = new 子类();
```

就是我们刚才说的多态写法！

##### 3. 特点

- 自动转换，**无需强转**
- 只能调用**父类有的方法**
- 运行执行**子类重写的方法**
- **不能调用子类独有的方法**（核心限制）

###### 代码示例

```java
Animal a = new Cat(); // 向上转型（自动）
a.eat(); // 能调用（父类有），执行猫的
// a.catchMouse(); 报错！子类独有方法不能调
```

#### 三、向下转型（强制转，有风险）

##### 1. 定义

**父类引用 转回 子类类型**

（大范围转小范围，**必须强转**）

##### 2. 格式

```java
子类 引用 = (子类) 父类引用;
```

##### 3. 作用

为了**调用子类独有的方法**！

###### 代码示例

```java
Animal a = new Cat(); // 向上转型
// 向下转型（强制）
Cat c = (Cat) a; 
c.catchMouse(); // 可以调用子类独有方法！
```

#### 四、巨大风险：类型转换异常

**只有原本就是子类对象的父类引用，才能转回子类！**

否则报错：`ClassCastException`

```java
Animal a = new Dog();
Cat c = (Cat) a; // 报错！狗不能强转成猫
```

##### 五、安全保障：instanceof 判断

转之前先判断，避免报错：

```java
if(a instanceof Cat) {
    Cat c = (Cat) a;
}
```

#### 六、一句话终极口诀

**上转自动，下转强转；上转不能调子类，下转必须是真身。**

#### 七、完整对比表

| 转型         | 方向    | 方式 | 风险 | 调用权限         |
| ------------ | ------- | ---- | ---- | ---------------- |
| **向上转型** | 子 → 父 | 自动 | 无   | 只能调父类方法   |
| **向下转型** | 父 → 子 | 强制 | 高   | 能调子类所有方法 |

#### 总结

1. **向上转型**：`Animal a = new Cat()`，**自动、安全**，不能调子类独有方法
2. **向下转型**：`Cat c = (Cat)a`，**强制、有风险**，能调子类独有方法
3. **判断**：用 `instanceof` 防止转换异常

### 封装：作用和实现方式

封装是 Java **面向对象三大特征之首**（封装、继承、多态），也是代码安全的核心。

---

#### 一、什么是封装？

**把属性私有化，对外提供公共的访问方法。**

就像：**把贵重物品锁进保险柜，只留一个钥匙给别人使用。**

#### 二、封装的核心作用

1. 安全性（最重要）

- 禁止外部直接修改属性
- 防止无效值（比如年龄设为 -100）

2. 易用性

- 对外只暴露简单方法，隐藏复杂实现
- 用户不用关心内部怎么写，直接用就行

3. 可维护性

- 内部代码随便改，不影响外部使用
- 降低耦合，方便后期维护

#### 三、封装的**实现方式**（固定 2 步）

##### 第一步：私有化属性（private）

不让外部直接访问

```java
private int age;
private String name;
```

##### 第二步：提供公共的 get/set 方法

给外部**安全访问**的入口

```java
// 读
public String getName() { return name; }

// 写（可以加逻辑判断）
public void setAge(int age) {
    if(age > 0 && age < 150) {
        this.age = age; // 合法才赋值
    }
}
```

#### 四、完整代码示例

```java
public class Student {
    // 1. 私有化属性
    private String name;
    private int age;

    // 2. 提供get/set方法
    public String getName() {
        return name;
    }

    public void setAge(int age) {
        // 封装的好处：可以加判断！
        if(age > 0) {
            this.age = age;
        }
    }

    public int getAge() {
        return age;
    }
}
```

#### 五、为什么要用封装？（对比一下就懂）

##### ❌ 不封装（不安全）

```java
student.age = -100; // 随便改，无效值，出bug
```

##### ✅ 封装后（安全）

```java
student.setAge(-100); // 无效，不会赋值
```

#### 六、总结

##### 封装是什么？

**私有化属性，提供公共的 get/set 方法。**

##### 封装作用？

1. **安全**（防止无效值、非法修改）
2. **隐藏实现**（易用）
3. **易维护**

##### 实现方式？

1. **private 修饰属性**
2. **提供 public 的 getXxx () /setXxx ()**

### String 不可变 & StringBuilder / StringBuffer 区别

#### 一、先搞懂：String 为什么不可变？

##### 1. 一句话结论

**String 是不可变字符串，每次修改都会产生新对象，效率低、浪费内存。**

##### 2. 底层原因（源码级）

- String 底层是 `private final char[] value`
- `final` 修饰 → 数组地址不能改
- `private` 修饰 → 外部无法修改内容

##### 3. 代码演示

```java
String s = "a";
s = s + "b";  // 看似修改，实际创建了新对象
```

⚠️ **+ 拼接字符串，循环中绝对禁止用 String！** 会产生大量垃圾对象，卡死程序。

#### 二、StringBuilder & StringBuffer（可变字符串）

它们底层**没有 final**，数组可直接扩容，**修改不产生新对象**，效率极高。

#### 三、三者核心区别表

| 对比         | String   | StringBuilder  | StringBuffer   |
| ------------ | -------- | -------------- | -------------- |
| **可变性**   | 不可变   | **可变**       | 可变           |
| **效率**     | 低       | **最高**       | 中             |
| **线程安全** | 安全     | **不安全**     | **安全**       |
| **适用场景** | 少量操作 | 单线程大量拼接 | 多线程大量拼接 |

#### 四、终极使用口诀

1. **少量字符串操作** → 用 **String**
2. **单线程大量拼接 / 修改** → 用 **StringBuilder**（最常用！）
3. **多线程并发操作** → 用 **StringBuffer**

#### 五、特殊记忆

1. String 为什么不可变？
   * 底层 `final char[]` 存储，私有且不可修改，**修改就产生新对象**。

2. StringBuilder 和 StringBuffer 区别？
   - **StringBuffer 线程安全（有 synchronized）**，效率低
   - **StringBuilder 非安全，效率最快**
   - 开发 99% 场景用 **StringBuilder**

3. 为什么循环拼接不能用 String？
   * 每次 `+` 都会创建新字符串对象，**内存爆炸、效率极低**。

#### 六、代码示例

```java
// 高效拼接（循环/大量字符串必用）
StringBuilder sb = new StringBuilder();
sb.append("a");
sb.append("b");
String result = sb.toString();
```

#### 总结

- **String**：不可变，效率低，少量用
- **StringBuilder**：可变，效率**最高**，单线程首选
- **StringBuffer**：可变，线程安全，多线程用
- 核心：**字符串频繁修改 → 绝对用 StringBuilder**
